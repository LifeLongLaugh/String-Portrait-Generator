What's New in v1.2

  • Precomputed line pixel maps for fast, memory-efficient scoring
  • Small-resolution L2 proxy prefilter for candidate shortlisting
  • Top-K shortlist + full-res L2 scoring for quality decisions
  • Occasional full global pass to recover filtered moves
  • No-Fear acceptance policy to escape local minima
  • Continuous stroke rendering with overlapping kernels
  • Enhanced UI with advanced controls and precompute status
  • Drag + wheel/pinch zoom for precise image placement

Implementation Notes

  • Uses large internal backing buffers for physical accuracy
  • Line pixel precomputation accelerates candidate scoring
  • Accuracy preserved by only altering scoring path, not commit path
  • Performance improvements focus on low-res scoring while maintaining high-res commit

Performance Considerations

The v1.2 engine includes significant optimizations but still requires substantial resources for high-quality results:

  • Precompute time: May take seconds for high nail counts & thickness (batched with progress UI)
  • Memory usage: Precomputed linePixels can use tens of MB for high nail counts
  • Processing power: Scoring optimized with Top-K shortlist and small-resolution prefilter
  • Resolution: Internal working resolution (default 1200, can increase to 1500 for finer detail)
