String Portrait Generator

v1.0 — Black thread | Physically-accurate canvas implementation

What this project does

The project focuses on numerical parity between what the algorithm removes from the brightness buffer and what it draws as thread using only vanilla JavaScript and plain HTML, CSS as backbone of the tool.

Quick start

  1. Open App to run the string-art generator.
  2. Load a photo.
  3. Fix an oval according to board.
  4. Position it under the fixed oval.
  5. Choose the number of nails and thread thickness.
  6. Generate the string-art portrait.
Open App View on GitHub

Included in v1.0

  • High-resolution backing buffer rendering (RES configurable in app.js).
  • Export thread sequence (.txt) and PNG of final artwork.
  • Simple UI: nail count, max strokes, thread thickness, reset, drag-to-position.

Design & implementation notes

  • The app.js uses large internal backing buffers (high RES) to preserve physical accuracy; expect high CPU use for large RES/nail counts.
  • linePixels precomputation is planned to accelerate candidate scoring — ideal for future versions.
  • Where performance is improved, accuracy is preserved by only altering the scoring path (low-res scoring vs high-res commit) — not the commit path itself.

How to contribute

  1. Fork the repository.
  2. Create a branch.
  3. Implement changes; keep parity and tests if possible.
  4. Open a pull request with a short description of changes and any performance tradeoffs.

Please keep new features opt-in (toggleable) and preserve the original behavior behind flags so v1.0 behavior is reproducible.
